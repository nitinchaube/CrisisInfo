# Event Intelligence Dashboard

A full-stack web application that analyzes social media tweets to extract and visualize event information using AI and natural language processing.

## Features

- **Tweet Analysis**: Submit tweets and extract event information using AI
- **Real-time Dashboard**: View events, statistics, and analytics
- **Interactive Graphs**: Visualize event networks and relationships
- **Advanced Filtering**: Filter events by category, location, and type
- **Modern UI**: Built with Material-UI and React

## Tech Stack

### Backend
- **Flask**: Python web framework
- **ChromaDB**: Vector database for embeddings
- **Sentence Transformers**: AI models for text processing
- **Gunicorn**: Production WSGI server

### Frontend
- **React**: JavaScript framework
- **Material-UI**: Component library
- **React Force Graph**: Interactive graph visualization
- **Axios**: HTTP client

## Deployment on Render

### Option 1: Using Render Blueprint (Recommended)

1. **Fork/Clone** this repository to your GitHub account

2. **Connect to Render**:
   - Go to [render.com](https://render.com)
   - Sign up/Login with your GitHub account
   - Click "New +" and select "Blueprint"

3. **Deploy with Blueprint**:
   - Connect your GitHub repository
   - Render will automatically detect the `render.yaml` file
   - Click "Apply" to deploy both services

4. **Environment Variables** (if needed):
   - The blueprint will set up most environment variables automatically
   - You can add additional ones in the Render dashboard

### Option 2: Manual Deployment

#### Backend Service

1. **Create Web Service**:
   - Name: `event-intelligence-backend`
   - Environment: `Python`
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `gunicorn app:app --bind 0.0.0.0:$PORT --workers 1 --timeout 120`

2. **Environment Variables**:
   ```
   PYTHON_VERSION=3.9.16
   FLASK_ENV=production
   SECRET_KEY=<auto-generated>
   CHROMA_DB_PATH=./chroma_db
   HUGGINGFACE_HUB_CACHE=./models
   TRANSFORMERS_CACHE=./models
   HF_HOME=./models
   ```

#### Frontend Service

1. **Create Static Site**:
   - Name: `event-intelligence-frontend`
   - Environment: `Static Site`
   - Build Command: `npm install && npm run build`
   - Publish Directory: `build`

2. **Environment Variables**:
   ```
   REACT_APP_API_BASE_URL=https://your-backend-service.onrender.com
   CI=false
   ```

## Local Development

### Prerequisites
- Python 3.9+
- Node.js 16+
- npm or yarn

### Backend Setup

1. **Navigate to backend directory**:
   ```bash
   cd backend
   ```

2. **Create virtual environment**:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

3. **Install dependencies**:
   ```bash
   pip install -r requirements.txt
   ```

4. **Run the server**:
   ```bash
   python app.py
   ```

### Frontend Setup

1. **Navigate to frontend directory**:
   ```bash
   cd frontend
   ```

2. **Install dependencies**:
   ```bash
   npm install
   ```

3. **Start development server**:
   ```bash
   npm start
   ```

## API Endpoints

### Events
- `GET /api/allEvents` - Get all events
- `GET /api/events` - Get filtered events
- `POST /api/submitTweet` - Submit tweet for analysis

### Metadata
- `GET /api/categories` - Get all categories
- `GET /api/locations` - Get all locations
- `GET /api/eventTypes` - Get all event types
- `GET /api/stats` - Get dashboard statistics

### Health
- `GET /api/health` - Health check endpoint

## Environment Variables

## Troubleshooting

### Common Issues

1. **Build Failures**:
   - Check Python version compatibility
   - Ensure all dependencies are in requirements.txt
   - Verify build commands in render.yaml

2. **API Connection Issues**:
   - Verify REACT_APP_API_BASE_URL is correct
   - Check CORS settings in backend
   - Ensure backend service is running

3. **Model Loading Issues**:
   - Models are downloaded on first use
   - Check HuggingFace Hub cache settings
   - Verify internet connectivity during build

### Performance Optimization

- Backend uses single worker to avoid memory issues
- Models are cached locally after first download
- Frontend is optimized for production builds
